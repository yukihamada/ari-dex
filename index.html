Why This Works

なぜ ARI は成功するのか

既存プレイヤーが解決できていない3つの構造的問題に、正しいタイミングで正しい解を提示する。

Core Thesis

DeFiの取引量は年間$1T超。しかし年間$10B以上がMEVとして搾取され、UXの悪さからユーザーの90%以上がCEXに留まっている。 根本原因は「注文の順序依存性」と「チェーンの分断」。 インテントベース + 暗号化バッチオークションはこれを構造的に解決する唯一のアーキテクチャであり、今まさにインフラが実用段階に達した。

Reason 1 — Problem

MEV問題は設計レベルで壊れている

Uniswapの注文は公開メモリプールを通過する。フロントランニングは原理的に防げない。Hookやプライベートメモリプールは「パッチ」。

ARI: 閾値暗号で注文を秘匿 → バッチ確定まで誰も中身を見れない → フロントランニングが物理的に不可能
Reason 2 — Opportunity

流動性の分断が勝者総取りの機会を生む

20以上のチェーンに流動性が分散。統一された体験を提供するプレイヤーはまだいない。「全部入り」は誰もやっていない。

ARI: インテントレイヤーがチェーンを抽象化。ユーザーは「ETHをUSDCに」と言うだけ。ソルバーが全チェーンから最適ルートを発見。
Reason 3 — Unfair Advantage

AIが開発コストの常識を破壊する

AIがコード生成を加速するが、暗号プロトコルのボトルネックは「数式の検証とセキュリティ担保」。開発費を圧縮した分を、外部監査・フォーマル検証・バグバウンティに全額投入。

ARI: AIでコード生成を10倍加速。ただし暗号・ZKの検証には外部専門家を必須配置。浮いた資金はセキュリティ監査（3社）とバグバウンティ（$10M）に全振り。

Core Innovation

Intent Engine

ユーザーは「意図」だけ伝える。最適な実行はプロトコルが保証する。

1

Intent Submission

「1 ETHを最良価格でUSDCに交換」— EIP-712準拠の署名付きインテントを送信

2

Threshold Encryption

閾値暗号（t-of-n BLS）で暗号化。バッチ確定まで誰も内容を閲覧不可

3

Solver Auction

ソルバーがブラインド入札で最良ルートを競争。CEX流動性も参加可能

4

Batch Decryption & Matching

250msバッチ終了後に一括復号。同一価格で約定（均一価格オークション）

5

Settlement

ZK証明付きでオンチェーン確定。トークンがウォレットに直接到着

Security

MEV Protection

年間$10B以上が搾取されるMEV問題を、パッチではなくアーキテクチャで解決

🔒 Encrypted Mempool

閾値暗号（t-of-n BLS）で注文内容を秘匿。バリデータの2/3が復号に参加しない限り不可視。サーチャーは何も見えない。

⚖ Batch Auction

250msバッチ内の全注文を均一価格で約定。注文の順序に意味がない = 先行者利益ゼロ。

💰 MEV Rebate

残余MEV（統計的アービトラージ等）の80%をユーザーに還元。残り20%がプロトコル収益。

Roadmap

Development Timeline

30ヶ月で完全分散化まで。暗号・ZKの検証フェーズに十分な時間を確保。

Phase 0 — Month 0-3

Foundation & Prototyping

設計確定、AIでRust workspace + CI/CDセットアップ。閾値暗号ライブラリの選定・PoC。暗号学アドバイザー契約。

Phase 1 — Month 3-10

Core Protocol + Testnet

ari-engine（バッチオークション）、ari-gateway、ari-crypto 実装。暗号プリミティブの数学的検証に4ヶ月。テストネットローンチ。

Phase 2 — Month 10-12

Solver Bootstrap

自社リファレンスソルバー稼働。Wintermute/GSR等の大手MM 2-3社とパイロット契約（初期は手数料ゼロ + 報酬プログラムで誘引）。最低限の流動性で価格競争力を実証。

Phase 3 — Month 12-16

Audit & Formal Verification

3社監査（Trail of Bits / OpenZeppelin / Spearbit）。Certora フォーマル検証。Immunefi $10Mバウンティ開始。修正→再監査サイクルに十分な期間。

Phase 4 — Month 16-20

Mainnet Launch (EVM)

Ethereum + Arbitrum + Base。$ARIトークンTGE。流動性マイニング開始。段階的にリミット引上げ（初月 TVL cap $50M）。

Phase 5 — Month 20-24

Multi-chain + Scale

Solana対応。クロスチェーンインテント有効化。ソルバーネットワーク拡大（10社+）。TVL $500M目標。

Phase 6 — Month 24-30

Full Decentralization

分散型シーケンサー移行。ガバナンス完全移譲。Guardian Multisig解散。プロトコル自律。

Solver Cold Start 戦略

Step 1: 自社リファレンスソルバー

初期はARI自身がソルバーとして稼働。主要DEX（Uniswap/Curve/Balancer）のオンチェーン流動性をルーティング。最低限の価格競争力を自前で保証。

Step 2: 大手MM招致

Wintermute/GSR/Flowに対し、初期6ヶ月は手数料ゼロ+ソルバー報酬プログラム（月$50K〜）を提供。実績あるMMの参加で価格改善を実証。

Step 3: ソルバーSDK公開

ari-solverをOSSで公開し、誰でもソルバーとして参加可能に。ソルバー数の増加 → 価格改善 → ユーザー増 → さらにソルバー参入のフライホイール。
