System Design

Architecture Overview

ARI is a 5-layer stack. Every layer is modular and independently upgradable.

Layer 5 — Frontend
React SPA, Mobile SDK, React Widget. Users express intents through a simple swap UI. No chain selection needed.
↓
Layer 4 — Gateway
REST + WebSocket API. Validates intents, rate-limits, and routes to the solver network. Handles Permit2 signature verification.
↓
Layer 3 — Solver Network
Competing solvers discover optimal execution routes across all connected chains. Dutch auction determines the winning solver for each batch.
↓
Layer 2 — Matching Engine
Encrypted batch auction. Threshold decryption reveals intents simultaneously. Uniform clearing price prevents MEV extraction.
↓
Layer 1 — Settlement
On-chain contracts (Ethereum, Arbitrum, Base). Settlement.sol executes fills atomically. Vault.sol manages CLMM liquidity positions as ERC-721 NFTs.

Become a Solver

Solvers are the backbone of ARI. They compete in real-time to find the best execution routes for user intents.

⚔

What Solvers Do

Solvers receive encrypted intent batches, compute optimal fill routes across AMMs, bridges, and private liquidity, then submit solutions to the matching engine. The best solution wins the batch.

⚡

Dutch Auction Mechanism

Each batch runs a Dutch auction: the price starts favorable for the solver and decays toward the user's limit. The first solver to submit a valid fill at the current price level wins. This incentivizes speed and price quality.

💰

Reward Model

Winning solvers earn 80% of trade fees for each batch they settle. The remaining 20% goes to the protocol treasury and $ARI stakers. Top solvers can earn additional performance bonuses.

How to Participate

1

Stake 100,000 $ARI

Solvers must stake a minimum of 100K $ARI tokens to the SolverRegistry contract. This stake acts as a performance bond and is slashable for malicious behavior (e.g., submitting invalid fills).

2

Run the Solver Node

Deploy ari-solver on your infrastructure. The node connects to the solver network, receives batch data, and submits solutions.

3

Build Your Strategy

Implement custom routing logic. Connect to on-chain AMMs, CEX APIs, private market makers, or cross-chain bridges. The solver SDK provides hooks for every step.

bash # Install and run the solver node cargo install ari-solver # Configure with your staking wallet and RPC endpoints ari-solver init \ --private-key $SOLVER_PRIVATE_KEY \ --rpc-ethereum $ETH_RPC_URL \ --rpc-arbitrum $ARB_RPC_URL \ --rpc-base $BASE_RPC_URL # Start solving ari-solver run --strategy custom --log-level info
Tooling

SDK & Tools

First-class support for Rust and TypeScript. Embed ARI swaps into any application.

Rust

ari-core

Core types, intent validation, cryptographic primitives (threshold encryption, Poseidon hashing). The foundational crate used by all other ARI packages.

toml [dependencies] ari-core = "0.1"
Rust

ari-solver

Full solver node implementation. Includes batch listener, route optimizer, solution builder, and on-chain settlement submitter. Extensible via trait-based strategy hooks.

toml [dependencies] ari-solver = "0.1"
Rust

ari-crypto

Threshold encryption (BLS12-381), Poseidon hash, zero-knowledge proof generation for private settlement. Used by both solvers and the matching engine.

toml [dependencies] ari-crypto = "0.1"
TypeScript

@ari-dex/sdk

TypeScript SDK for browser and Node.js. Submit intents, stream market data, manage Permit2 signatures. Full type safety with auto-generated types from the Rust core.

bash npm install @ari-dex/sdk

On-Chain

Smart Contracts

Minimal on-chain footprint. All complex logic runs off-chain in solvers; contracts only handle atomic settlement and liquidity custody.

Solidity

Settlement.sol

Executes solver-submitted fills atomically. Uses Permit2 for gasless token approvals. Verifies solver signatures, enforces deadline + slippage constraints, and distributes fees.

Key functions: settle(), batchSettle(), cancelIntent()
Solidity

Vault.sol

Concentrated Liquidity Market Maker (CLMM) vault. LP positions are represented as ERC-721 NFTs, each encoding a tick range and fee tier. Supports single-sided deposits and auto-compounding.

Key functions: mint(), burn(), collect(), rebalance()
Solidity

SolverRegistry.sol

Manages solver staking, slashing, and reputation scores. Solvers must maintain a minimum 100K $ARI stake. Slashing conditions: invalid fills, timeout violations, or provable collusion.

Key functions: register(), slash(), withdraw()

Data Model

Intent Schema

The canonical Rust type representing a user intent. This is the core data structure that flows through every layer of the stack.

rust use ethers::types::{Address, U256, Bytes}; use serde::{Serialize, Deserialize}; /// A user's trade intent — the atomic unit of the ARI protocol. /// Intents are encrypted before entering the mempool and only /// revealed via threshold decryption at batch settlement time. #[derive(Debug, Clone, Serialize, Deserialize)] pub struct Intent { /// Unique intent identifier (keccak256 hash of fields) pub id: [u8; 32], /// The user's wallet address (signer) pub user: Address, /// Token being sold pub sell_token: Address, /// Token being bought pub buy_token: Address, /// Exact amount of sell_token to spend (in wei) pub sell_amount: U256, /// Minimum acceptable buy_token received (slippage protection) pub min_buy_amount: U256, /// Source chain ID (e.g., 1 = Ethereum, 42161 = Arbitrum) pub source_chain: u64, /// Destination chain ID (can differ from source for cross-chain) pub dest_chain: u64, /// Unix timestamp — intent expires after this time pub deadline: u64, /// Nonce to prevent replay attacks pub nonce: u64, /// Intent type: Swap, LimitOrder, TWAP, or DCA pub intent_type: IntentType, /// Permit2 signature for gasless token approval pub permit2_signature: Bytes, /// Threshold-encrypted payload (decrypted only at batch time) pub encrypted_payload: Option<Bytes>, } #[derive(Debug, Clone, Serialize, Deserialize)] pub enum IntentType { /// Market swap — fill at best available price Swap, /// Limit order — fill only at specified price or better LimitOrder { limit_price: U256 }, /// Time-weighted average price over N batches TWAP { num_batches: u32, interval_secs: u64 }, /// Dollar-cost averaging over a schedule DCA { frequency_secs: u64, total_epochs: u32 }, } #[derive(Debug, Clone, Serialize, Deserialize)] pub enum IntentStatus { Pending, // Submitted, awaiting batch inclusion Encrypted, // In encrypted mempool Matched, // Solver found a fill route Settling, // On-chain settlement tx submitted Filled, // Successfully settled Expired, // Deadline passed without fill Cancelled, // User cancelled before batch close }
